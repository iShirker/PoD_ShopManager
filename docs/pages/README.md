# Page specifications

Each `.md` file describes one app page (or screen). Filename matches the page component (e.g. `Dashboard.md` → `Dashboard.tsx`).

**Sections:**
- **Goals** — What the screen achieves.
- **Feature research** — User needs, competitor patterns, forums.
- **UI design** — Layout, key elements, actions.
- **Functional tests** — Verifiable checks for correct/incorrect behaviour.

See [IMPLEMENTATION_PLAN.md](../../IMPLEMENTATION_PLAN.md) and [UI_MENU_STRUCTURE.md](../../UI_MENU_STRUCTURE.md).
